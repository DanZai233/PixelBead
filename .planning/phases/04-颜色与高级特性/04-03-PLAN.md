<task type="auto">
  <name>Task 2: Create image processing utilities</name>
  <files>
    apps/mobile/src/utils/imageProcessor.ts
  </files>
  <action>
    Create apps/mobile/src/utils/imageProcessor.ts:

    1. Export functions:
       - pixelateImage(imageUri: string, targetWidth: number, targetHeight: number, palette?: PaletteColor[]): Promise<Map<string, string>>
       - cropImage(imageUri: string, cropRegion: { x, y, width, height }): Promise<string> (returns cropped image URI)
       - removeBackground(imageUri: string, threshold: number = 0.1): Promise<Map<string, string>>
       - excludeColors(grid: Map<string, string>, excludeHexes: string[]): Map<string, string>
    2. pixelateImage implementation:
       - Load image using Image.getSize() (React Native API)
       - Create canvas (offline, use Expo ImageManipulator or native canvas)
       - Resize image to target dimensions (nearest-neighbor sampling, no anti-aliasing)
       - For each pixel: Get RGB color, find closest in palette (if provided), add to Map with key "x,y"
       - Return Map<string, string> grid
    3. cropImage implementation:
       - Use Expo ImageManipulator.crop() if available
       - Or create custom crop by sampling only pixels within crop region
    4. removeBackground implementation:
       - Analyze corner pixels to detect background color
       - Use colorDistanceHSL from @pixelbead/color-system to find similar colors within threshold
       - Return new Map without background pixels
    5. excludeColors implementation:
       - Filter grid Map to remove pixels matching excludeHexes
       - Return filtered Map

    Performance: Use chunked processing for large images (100 pixels per frame) to avoid UI blocking.
  </action>
  <verify>
    <automated>npx tsc --noEmit --project apps/mobile/tsconfig.json</automated>
  </verify>
  <done>
    imageProcessor exports all functions, TypeScript compiles, functions handle edge cases (null images, empty grids)
  </done>
</task>
