  <name>Task 1: Create share service with Redis integration</name>
  <files>apps/mobile/src/services/shareService.ts</files>
  <action>Create service for share link generation and loading via Upstash Redis:

1. Import:
   - ColorHex, PixelStyle from @pixelbead/shared-types

2. Define constants:
   const REDIS_URL = 'https://pixelbead.upstash.io'
   const SHARE_TTL_SECONDS = 604800 // 7 days
   const SCHEME = 'pixelbead'

3. Define ShareData interface:
   ```typescript
   interface ShareData {
     name: string;
     grid: ColorHex[][];
     gridWidth: number;
     gridHeight: number;
     pixelStyle: string;
     createdAt: number;
   }
   ```

4. Generate unique share ID:
   - generateShareId(): return random string (16 chars) using Math.random().toString(36).substr(2, 16)

5. Implement generateShareLink(projectData):
   - Input: { name, grid, gridWidth, gridHeight, pixelStyle }
   - Convert pixelStyle to string: pixelStyle.toString()
   - Create ShareData object with createdAt: Date.now()
   - Generate share ID: shareId = generateShareId()
   - Redis key: `share:${shareId}`
   - POST to `${REDIS_URL}/set/${key}`
     - Headers: { Authorization: `Bearer ${REDIS_TOKEN}`, 'Content-Type': 'application/json' }
     - Body: JSON.stringify(shareData)
     - Query params: ex=${SHARE_TTL_SECONDS} (TTL)
   - On success: Return share URL: `${SCHEME}://share/${shareId}`
   - On error: throw Error("Failed to generate share link")

6. Implement loadSharedProject(shareId):
   - Input: shareId (string)
   - Redis key: `share:${shareId}`
   - GET from `${REDIS_URL}/get/${key}`
     - Headers: { Authorization: `Bearer ${REDIS_TOKEN}` }
   - On success: Parse response as ShareData
   - Convert pixelStyle string back to PixelStyle enum
   - Return ShareData object
   - On 404: throw Error("Share link expired or not found")
   - On error: throw Error("Failed to load shared project")

7. Error handling:
   - Try/catch around fetch calls
   - Parse error responses from Redis
   - Re-throw with descriptive messages

8. Environment variables:
   - REDIS_TOKEN from process.env (not available in mobile app)
   - Note: Mobile apps need different approach for secrets
   - Option 1: Hardcode (insecure)
   - Option 2: Use Expo Constants (expo-constants) with app.json config
   - Option 3: Fetch from secure endpoint (web app proxy)
   - Recommendation: Use web app proxy endpoint or store in app.json (not ideal)

   For this plan, assume REDIS_TOKEN is available via Constants.expoConfig.extra.redisToken (configure in app.json).

Export generateShareLink, loadSharedProject, generateShareId.
</action>
