         ```typescript
           },
           setItem: (name: string, value: string): void => {

           },
           removeItem: (name: string): void => {
             storage.delete(name)
           },
         }
         ```

         ```typescript
         export function createPersistedStore<T>(
           name: string,
           initialState: T
         ) {
           return create<T>()(
             persist(
               () => initialState,
               {
                 name,
                 storage: createJSONStorage(() => mmkvStorage),
               }
             )
           )
         }
         ```

    Rationale: MMKV persistence middleware automatically saves state changes (per D-05). Custom adapter bridges Zustand's storage interface to MMKV. JSON serialization via createJSONStorage handles complex data structures.

         - import { ToolType } from '@pixelbead/shared-types'
         - import { PixelStyle } from '@pixelbead/shared-types'

       - Define CanvasState interface (modular store organization, per D-04):
         ```typescript
         interface CanvasState {
           // Grid state
           grid: ColorHex[][]
           gridSize: number

           // Tool selection
           selectedTool: ToolType
           brushSize: number
           selectedColor: ColorHex

           // Drawing state
           isDrawing: boolean

           // History (undo/redo)
           undoStack: string[] // JSON-serialized grids
           redoStack: string[]

           // Actions
           setGrid: (grid: ColorHex[][]) => void
           setSelectedTool: (tool: ToolType) => void
           setSelectedColor: (color: ColorHex) => void
           setBrushSize: (size: number) => void
           pushToUndoStack: (grid: ColorHex[][]) => void
           undo: () => void
           redo: () => void
           clearCanvas: () => void
         }
         ```

       - Create initial state:
         ```typescript
         const initialCanvasState: CanvasState = {
           grid: Array(20).fill(null).map(() => Array(20).fill('#FFFFFF')),
           gridSize: 20,
           selectedTool: ToolType.PENCIL,
           brushSize: 1,
           selectedColor: '#000000',
           isDrawing: false,
           undoStack: [],
           redoStack: [],
           // Actions will be defined below
         }
         ```

       - Implement actions with type safety:
         - setGrid: updates grid state
         - setSelectedTool: updates tool selection
         - setSelectedColor: updates selected color
         - setBrushSize: updates brush size (1-5 pixels, per DRAW-08)
         - pushToUndoStack: serializes grid to JSON, pushes to undoStack
         - undo: pops from undoStack, deserializes to grid, pushes current grid to redoStack
         - redo: pops from redoStack, deserializes to grid, pushes current grid to undoStack
         - clearCanvas: resets grid to empty

       - Create store with MMKV persistence:
         ```typescript
         export const useCanvasStore = create<CanvasState>()(
           persist(
             (set) => ({
               ...initialCanvasState,
               // Actions here
             }),
             {
               name: 'canvas-storage',
               storage: createJSONStorage(() => mmkvStorage),
               partialize: (state) => ({
                 // Persist only these fields (don't persist isDrawing, redoStack)
                 grid: state.grid,
                 gridSize: state.gridSize,
                 selectedTool: state.selectedTool,
                 brushSize: state.brushSize,
                 selectedColor: state.selectedColor,
                 undoStack: state.undoStack,
               }),
             }
           )
         )
         ```

       - Note: partialize option controls which state is persisted (redoStack not persisted per STAT-04: Undo/redo history persists within session)

    Rationale: Modular store organization separates concerns (per D-04). Canvas-specific state isolated from settings and UI. Selector-based subscriptions prevent unnecessary re-renders (per D-06). MMKV persistence saves canvas state across app restarts (per STAT-02).
  </action>
