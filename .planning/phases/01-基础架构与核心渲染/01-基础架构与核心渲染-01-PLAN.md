<task type="auto">
  <name>Task 1: Convert to pnpm workspace and create monorepo structure</name>
  <files>pnpm-workspace.yaml, package.json</files>
  <action>
    1. Create pnpm-workspace.yaml at root with workspace configuration:
       - packages: ['apps/*', 'packages/*']
       - This defines monorepo structure (per D-01: Expo SDK 52+ automatic monorepo with pnpm workspaces)

    2. Update root package.json to enable pnpm workspace:
       - Add "private": true (already exists)
       - Remove devDependencies.lockfileVersion if present (pnpm generates its own lockfile)
       - Keep existing web app dependencies (they're for the root app)

    3. Create directory structure:
       - mkdir -p apps
       - mkdir -p packages

    4. Install pnpm if not already installed:
       - npm install -g pnpm

    5. Remove npm lockfile and install with pnpm:
       - rm package-lock.json
       - pnpm install (this generates pnpm-lock.yaml)

    Rationale: Expo monorepo docs recommend pnpm workspaces for automatic Metro bundler configuration. Converting from npm to pnpm enables workspace protocol dependencies and hoisting control (per RESEARCH.md stack recommendation).
  </action>
  <verify>
    <automated>test -f pnpm-workspace.yaml && test -f pnpm-lock.yaml && test ! -f package-lock.json</automated>
  </verify>
  <done>pnpm workspace configuration exists with apps/* and packages/* patterns, pnpm-lock.yaml generated, npm lockfile removed</done>
</task>

<task type="auto">
  <name>Task 2: Initialize Expo SDK 55 mobile app in apps/mobile</name>
  <files>apps/mobile/package.json, apps/mobile/app.json, apps/mobile/tsconfig.json</files>
  <action>
    1. Initialize Expo app with automatic TypeScript and New Architecture:
       - cd apps
       - npx create-expo-app@latest mobile --template blank-typescript --yes
       - This creates apps/mobile with Expo SDK 55 (per D-12: Use Expo SDK 55 with New Architecture enabled)

    2. Modify apps/mobile/package.json:
       - Ensure "expo": "~55.0.0" or compatible SDK 55 version
       - Add "expo-router": "^4.0.0" for file-based routing (per D-13: Use Expo Router for file-based routing)
       - Add "expo-dev-client": "~5.0.0" for development builds
       - Add dependencies needed for monorepo:
         - "typescript": "^5.0.0"
         - "@types/react": "~19.0.0"
         - "@types/react-native": "~0.74.0"

    3. Verify apps/mobile/app.json has New Architecture enabled:
       - Ensure "newArchEnabled": true (per D-10: Initialize MMKV with New Architecture enabled)

    4. Configure TypeScript in apps/mobile/tsconfig.json:
       - Enable strict mode: "strict": true (per D-08: TypeScript strict mode enabled)
       - Configure paths for workspace: add to compilerOptions.paths:
         - "@pixelbead/*": ["../../packages/*/src"]
         - This enables path alias imports (per D-14: Use path aliases for shared package imports)

    5. Clean up default Expo template files:
       - Remove assets/background.png if not needed
       - Keep app/_layout.tsx for Expo Router configuration

    Rationale: Expo SDK 55 is the latest stable version with New Architecture support (React Native 0.84+). New Architecture required for MMKV v4 JSI bindings (critical for synchronous storage performance per PITFALLS.md).
  </action>
  <verify>
    <automated>cd apps/mobile && pnpm list expo | grep "55"</automated>
  </verify>
  <done>Expo SDK 55 mobile app initialized with TypeScript strict mode, New Architecture enabled, Expo Router configured, path aliases set up</done>
</task>
